"Et tu, Brute?"

The Caesar Cipher is a 2,000-year-old shift cipher — every letter rotates forward by n positions in the alphabet. Below is a live JavaScript port of the same algorithm I wrote in Java. Type below, rotate the wheel, watch the letters travel.

§ The algorithm

Each letter is treated as a position on a 26-character ring. To encrypt, we move each letter forward by shift positions, wrapping back to A after Z. Non-letters pass through untouched. Decryption is the same operation with the opposite shift — so we don't write two algorithms, just one.

The Java implementation uses StringBuilder for efficiency, normalizes the shift to [0, 25] with a modulo trick that handles negatives, and preserves case. The JavaScript on this page does the same thing, in roughly the same shape.

// CaesarsCipher.java — the core method
public String encrypt(String text, int shift) {
    int n = ((shift % 26) + 26) % 26;
    StringBuilder out = new StringBuilder(text.length());

    for (char c : text.toCharArray()) {
        if (Character.isLetter(c)) {
            char base = Character.isUpperCase(c) ? 'A' : 'a';
            out.append((char) (base + (c - base + n) % 26));
        } else {
            out.append(c);
        }
    }
    return out.toString();
}
§ Honest note

Caesar is, of course, trivially breakable. Brute-force all 26 shifts and you'll spot the readable one in a second; frequency-analyse the output and you'll spot it even faster. This implementation is for learning — about modular arithmetic, character codes, and writing a clean, well-documented Java class — not for protecting anything that matters. For real secrecy, use AES, ChaCha20, or anything else maintained by people who think about side channels for a living.
